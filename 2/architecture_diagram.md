# Архітектура системи "Clothify Store"

Цей документ описує технічну структуру додатка та взаємодію між його основними компонентами. Система побудована за класичною трирівневою архітектурою (**Client-Server-Database**).

---

## 1. Основні блоки системи

###  Client (Frontend)
Це частина системи, з якою безпосередньо взаємодіє користувач у браузері.
* **Технології:** HTML5, CSS3, JavaScript (Vanilla JS).
* **Функції:** Відображення інтерфейсу, валідація введених даних на стороні клієнта, надсилання асинхронних запитів до сервера через **Fetch API**.
* **Стан:** Використовує `localStorage` для збереження активної сесії користувача без використання складних стейт-менеджерів.

###  Backend (Flask API)
Центральний вузол логіки додатка, який обробляє всі запити від клієнта.
* **Технології:** Python, Flask.
* **Функції:**
    * **Маршрутизація (Routing):** Обробка URL-запитів та повернення відповідних HTML-сторінок або JSON-даних.
    * **Авторизація:** Перевірка прав доступу та безпечне хешування паролів.
    * **Business Logic:** Обробка операцій з кошиком (групування товарів x2/x3) та управління профілем.

###  Database (SQLite)
Шар збереження даних додатка.
* **Технології:** SQLite (через SQLAlchemy ORM).
* **Структура:** Містить таблиці для користувачів (`User`), товарів (`Item`) та записів кошика (`Cart`).
* **Перевага:** Файлова база даних не потребує окремого сервера, що ідеально підходить для швидкої розробки та навчання.

---

## 2. Рух даних (Data Flow)

Процес взаємодії між компонентами під час типової дії користувача:

1.  **Запит:** Користувач натискає "У кошик". Клієнтський скрипт надсилає `POST`-запит із `item_id` на сервер.
2.  **Обробка:** Flask-сервер отримує запит, ідентифікує користувача та звертається до бази даних через SQLAlchemy.
3.  **Збереження:** БД оновлює кількість (`quantity`) товару в таблиці `Cart`.
4.  **Відповідь:** Сервер повертає JSON-підтвердження. Фронтенд оновлює інтерфейс користувача без перезавантаження сторінки.

## 3. Схема архітектури

> **Примітка:** Для візуалізації цієї архітектури використовується стандартна схема взаємодії клієнт-сервер.
