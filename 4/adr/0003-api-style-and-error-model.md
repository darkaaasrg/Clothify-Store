# ADR 0003: Стиль REST та модель помилок

## Контекст
Необхідно забезпечити єдиний стандарт взаємодії між фронтендом та бекендом.

## Рішення
1. **Стиль**: Використання стандартних HTTP-методів (GET, POST, PUT, DELETE) згідно з принципами REST.
2. **Формат помилок**: Помилки повертаються у JSON форматі з ключами `error` та `code`.
3. **Політика кодів**:
   - `201 Created`: Успішне створення (напр. відгуку).
   - `204 No Content`: Успішне видалення.
   - `400 Bad Request`: Помилка валідації даних.
   - `404 Not Found`: Ресурс не знайдено або помилка бізнес-логіки.

## Вимога відповідності
Усі ендпоїнти повинні суворо відповідати специфікації `openapi.yaml`.
