# Технічний опис моделей (domain/models.py)

Цей документ містить детальний опис структури сутностей проєкту **Clothify Store**, розроблених згідно з принципами **Domain-Driven Design (DDD)**.

## 1. Піддомен Користувачів (User Subdomain)
**Сутність: `User`**
* **Опис:** Зберігає ідентичність користувача та дані його профілю.
* **Поля:**
    * `id` (Integer): Унікальний первинний ключ.
    * `username` (String): Унікальний логін/email для входу в систему.
    * `name` (String): Публічне ім'я користувача.
    * `password` (String): Хешований рядок пароля для безпеки.
    * `avatar` (String): Шлях до файлу зображення профілю (за замовчуванням `default_avatar.png`).

---

## 2. Піддомен Каталогу (Catalog Subdomain)
**Сутність: `Item` (Товар)**
* **Опис:** Містить повний опис характеристик одягу в межах домену каталогу.
* **Поля:**
    * `id` (Integer): Унікальний ідентифікатор товару.
    * `name` (String): Назва моделі одягу.
    * `price` (Integer): Вартість товару в гривнях.
    * `category` (String): Групування за типом (наприклад, "Верх", "Низ").
    * `image_url` (String): Посилання на зображення товару.
    * `fabric` (String): Склад тканини (ключова бізнес-характеристика).
    * `origin` (String): Країна виробництва.
    * `description` (Text): Розширений опис товару.

**Сутність: `Review` (Відгук)**
* **Опис:** Реалізує соціальний зв'язок між користувачем та товаром.
* **Поля:**
    * `item_id` (Integer): Зовнішній ключ до товару.
    * `user_id` (Integer): Зовнішній ключ до автора відгуку.
    * `text` (Text): Текстовий зміст відгуку.
* **Зв'язки:**
    * `user`: Використовує **Ubiquitous Language** для доступу до даних автора (ім'я, аватар) через зв'язок `db.relationship`.

---

## 3. Піддомен Кошика та Замовлень (Ordering Subdomain)
**Сутність: `Cart` (Кошик)**
* **Опис:** Відповідає за бізнес-логіку підготовки товарів до покупки.
* **Поля:**
    * `user_id` / `item_id`: Зв'язок між користувачем та обраним товаром.
    * `quantity` (Integer): Кількість одиниць товару (за замовчуванням 1).
* **Мінімальні методи (Прототипування):**
    * `add_item_to_cart(user_id, item_id)`: Статичний метод, що реалізує бізнес-правило групування. Якщо товар вже існує в кошику користувача, метод не створює новий запис, а інкрементує поле `quantity` (реалізація логіки x2/x3).
