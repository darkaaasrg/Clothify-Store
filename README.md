Технологічний стек
- **Backend:** Python 3.13, Flask.
- **Database:** SQLite (SQLAlchemy ORM).
- **Frontend:** HTML5, CSS3 (Glassmorphism, Flexbox, Grid), JavaScript (Vanilla JS).
- **Security:** Хешування паролів через Werkzeug.

### Архітектура та Resilience (Надійність)

Проєкт реалізований з урахуванням сучасних патернів відмовостійкості, що робить його стійким до високих навантажень та мережевих збоїв:

* **Idempotency Strategy**: Використання заголовка `Idempotency-Key` для всіх критичних операцій (наприклад, додавання у кошик). Це гарантує, що навіть при повторному надсиланні запиту через лаги мережі, дані в базі не дублюватимуться.
* **Request Tracing**: Кожен запит маркується унікальним `X-Request-Id`. Це дозволяє "прошити" логіку від клієнта до сервера і швидко знайти помилку в логах за конкретним ідентифікатором.
* **Layered Design**:
    * **Domain Layer**: опис сутностей та логіки бази даних (SQLAlchemy).
    * **Service Layer**: чиста бізнес-логіка (валідація, розрахунки).
    * **API Layer**: Flask-маршрути, що відповідають за HTTP-статуси та JSON-відповіді.
* **Client-Side Resilience**: Фронтенд нативно обробляє помилки сервера (429, 500) та реалізує стратегію **Degraded Mode** — сайт залишається доступним для перегляду навіть при тимчасових збоях бекенду.
